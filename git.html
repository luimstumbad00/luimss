¿QUE ES?

Git es un sistema de control de versiones distribuido que realiza un seguimiento de los cambios en cualquier conjunto de archivos de computadora, generalmente se usa para coordinar el trabajo entre programadores que desarrollan código fuente en colaboración durante el desarrollo de software.

COMANDOS MAS UTILIZADOS:

1-NIT

2-STATUS

3-BARNCH

4-CHECKOUT

5-PULL

6-PUSH

7-CLONE

8-COMMIT

9-ADD

1-INIT

Es un comando que se utiliza una sola vez durante la configuración inicial de un repositorio nuevo. Al ejecutar este comando, se creará un nuevo subdirectorio .

2-STATUS

Muestra el estado del directorio de trabajo y el área de ensayo . Le permite ver qué cambios se han realizado, cuáles no y qué archivos no están siendo rastreados por Git.

3-BRANCH

Te permite crear, enumerar y eliminar ramas, así como cambiar su nombre. No te permite cambiar entre ramas o volver a unir un historial bifurcado.

4-CHECKOUT

Te permite desplazarte entre las ramas creadas por git branch .

5-PULL

Se emplea para extraer y descargar contenido desde un repositorio remoto y actualizar al instante el repositorio local para reflejar ese contenido.

6-PUSH

Se usa para cargar contenido del repositorio local a un repositorio remoto. El envío es la forma de transferir confirmaciones desde tu repositorio local a un repositorio remoto.

7-CLONE

Se utiliza principalmente para apuntar a un repositorio existente y clonar o copiar dicho repositorio en un nuevo directorio, en otra ubicación.

8-COMMIT

Captura una instantánea de los cambios preparados en ese momento del proyecto. Las instantáneas confirmadas pueden considerarse como versiones "seguras" de un proyecto: Git no las cambiará nunca a no ser que se lo pidas expresamente.

9-ADD

Añade un cambio del directorio de trabajo en el entorno de ensayo. De este modo, indica a Git que quieres incluir actualizaciones en un archivo concreto en la próxima confirmación.
